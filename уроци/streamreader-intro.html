StreamReader

Как програмите четат информация от файлове? Днес ще се научим да „отваряме книга" с помощта на поток — като тръба за данни между файла и програмата.

01 КАКВО Е „ПОТОК ОТ ДАННИ"?

Досега данните в нашите програми живееха само в променливи — и изчезваха, щом програмата приключи. Но какво ако искаме да прочетем нещо от файл?

◎ АНАЛОГИЯ · ВОДОПРОВОДНА ТРЪБА

Представи си, че файлът е водоем, а програмата ти — чаша. Потокът (stream) е тръбата, която ги свързва и позволява водата (данните) да тече.

Ключова идея: Потокът е връзка между програмата ти и външен източник на данни — файл, мрежа, клавиатура. Днес ще работим с файлове.

02 ЗАЩО ДА ЧЕТЕМ ОТ ФАЙЛОВЕ?

⌨ Console.ReadLine()

Потребителят въвежда данните ръчно всеки път. Бавно, неудобно за много данни.

▤ StreamReader

Програмата чете данните от файл автоматично. Бързо, удобно, повтаряемо.

◈ ПРИМЕРИ ОТ РЕАЛНИЯ ЖИВОТ

  1. Списък с ученици — програмата чете имената от файл вместо да ги питаш един по един
  2. Настройки на игра — играта зарежда запазени настройки при стартиране
  3. Данни от сензор — анализираш температурни измервания, записани във файл
  4. Обработка на текст — преброяваш думите в роман, без да ги пишеш ръчно

03 StreamReader — 3 СТЪПКИ

StreamReader е клас в C#, който ни позволява да четем текст от файл — ред по ред или целия наведнъж.

1

ОТВОРИ

StreamReader sr = new StreamReader( "data.txt");
2

ПРОЧЕТИ

string line = sr.ReadLine(); // или ReadToEnd()
3

ЗАТВОРИ

sr.Close(); // освободи // ресурса!
◎ АНАЛОГИЯ · КРАНЧЕ

1. Отваряш кранчето (създаваш StreamReader)
2. Пълниш чашата (четеш данните)
3. Затваряш кранчето (затваряш StreamReader) — задължително!

▲

Важно: Винаги затваряй StreamReader след използване! Иначе файлът остава „заключен" и други програми не могат да го използват.

04 ПРИМЕР · ЧЕТЕНЕ РЕД ПО РЕД

Нека имаме файл data.txt със следното съдържание:

data.txt
Мария Иванова
Петър Георгиев
Елена Димитрова

Ето как го четем в C#:

Program.cs
using System.IO;  // Нужно за StreamReader!

static void Main()
{
    // Стъпка 1: Отваряме файла
    StreamReader sr = new StreamReader("data.txt");

    // Стъпка 2: Четем ред по ред
    string line1 = sr.ReadLine();  // "Мария Иванова"
    string line2 = sr.ReadLine();  // "Петър Георгиев"
    string line3 = sr.ReadLine();  // "Елена Димитрова"

    Console.WriteLine(line1);
    Console.WriteLine(line2);
    Console.WriteLine(line3);

    // Стъпка 3: Затваряме!
    sr.Close();
}
как работи ReadLine() · показалецът слиза надолу
▶ Мария Иванова
→ 1. ReadLine() = "Мария Иванова"
▶ Петър Георгиев
→ 2. ReadLine() = "Петър Георгиев"
▶ Елена Димитрова
→ 3. ReadLine() = "Елена Димитрова"
✪

Забележка: ReadLine() връща null, когато файлът свърши. Така разбираме, че няма повече редове.

05 ЧЕТЕНЕ С ЦИКЪЛ WHILE

Когато не знаем колко реда има файлът, използваме цикъл:

Program.cs
using System.IO;

static void Main()
{
    StreamReader sr = new StreamReader("data.txt");

    string line = sr.ReadLine();       // първия ред

    while (line != null)          // докато има редове
    {
        Console.WriteLine(line);    // отпечатай
        line = sr.ReadLine();        // прочети следващия
    }

    sr.Close();
}
как работи цикълът
Прочети първия ред
▼
line != null ?
▼ ДА
WriteLine(line)
line = sr.ReadLine()
↻ обратно към проверката
— — —
НЕ → ▼
sr.Close() → край
✓

Шаблон: Този шаблон с while цикъл ще използваш постоянно. Запомни го — работи за файлове с произволен брой редове.

06 ReadToEnd() · ЦЕЛИЯТ ФАЙЛ НАВЕДНЪЖ

Ако искаме целия текст наведнъж (без да го четем ред по ред):

Program.cs
using System.IO;

static void Main()
{
    StreamReader sr = new StreamReader("data.txt");

    string all = sr.ReadToEnd();  // всичко наведнъж!
    Console.WriteLine(all);

    sr.Close();
}
▤ ReadLine()

Чете един ред. Подходящо за обработка ред по ред — търсене, броене, филтриране.

▣ ReadToEnd()

Чете целия файл като един голям string. По-просто, но по-трудно за обработка.

07 БОНУС · using { } — ПО-БЕЗОПАСЕН НАЧИН

Какво става ако програмата гръмне с грешка преди да стигне до sr.Close()? Файлът остава заключен. Лошо.

Решението е using блок — той автоматично затваря StreamReader, дори при грешка:

Program.cs — по-безопасен
using System.IO;

static void Main()
{
    using (StreamReader sr = new StreamReader("data.txt"))
    {
        string line = sr.ReadLine();
        while (line != null)
        {
            Console.WriteLine(line);
            line = sr.ReadLine();
        }
    } // ← Тук sr се затваря АВТОМАТИЧНО!
}
◎ АНАЛОГИЯ · КРАН С ТАЙМЕР

using блокът е като кран, който сам се затваря след употреба. Не можеш да забравиш да го затвориш — системата се грижи за теб.

◎

Съвет: Използвай using блока винаги, когато е възможно. Това е добра практика в C#.

08 ОБОБЩЕНИЕ НА УРОКА

01

using System.IO;

Пространство от имена, нужно за StreamReader

02

ReadLine()

Чете файла ред по ред · идеално с while цикъл

03

ReadToEnd()

Чете целия файл наведнъж · като един string

04

using { ... }

Автоматично затваря ресурса · безопасна практика

  1. Потокът е връзката между програмата и файла — като тръба за данни.
  2. StreamReader чете текстови файлове. Винаги: отвори → прочети → затвори.
  3. ReadLine() чете по един ред. Връща null, когато файлът свърши.
  4. ReadToEnd() чете целия файл като един string.
  5. using блокът гарантира, че StreamReader ще бъде затворен автоматично.
