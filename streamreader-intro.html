Потоци от данни
в C#

Как програмите четат информация от файлове? Днес ще научим как да „отворим книга" с помощта на StreamReader.

🤔 Какво е „поток от данни"?

Досега данните в нашите програми живееха само в променливи — и изчезваха, когато програмата приключи. Но какво ако искаме да прочетем нещо от файл?

Аналогия: Водопроводна тръба 🚰

Представете си, че файлът е водоем, а вашата програма е чаша. Потокът (stream) е тръбата, която свързва водоема с чашата и позволява водата (данните) да тече.

📌
Ключова идея: Потокът е връзка между вашата програма и външен източник на данни (файл, мрежа, клавиатура). Днес ще работим с файлове.

📁 Защо да четем от файлове?

⌨️
Console.ReadLine()

Потребителят въвежда данните ръчно всеки път. Бавно, неудобно за много данни.

📄
StreamReader

Програмата чете данните от файл автоматично. Бързо, удобно, многократно!

Примери от реалния живот:

  • Списък с ученици — програмата чете имената от файл вместо да ги питате един по един
  • Настройки на игра — играта зарежда запазени настройки от файл при стартиране
  • Данни от сензор — програма анализира температурни измервания, записани във файл

🔧 StreamReader — стъпка по стъпка

StreamReader е клас в C#, който ни позволява да четем текст от файл ред по ред или целия наведнъж.

⚠️
Важно! Винаги затваряйте StreamReader след използване! Ако забравите, файлът остава „заключен" и други програми не могат да го използват.

📖 Пример 1: Четене ред по ред

Нека имаме файл data.txt със следното съдържание:

data.txt 
Мария Иванова
Петър Георгиев
Елена Димитрова

Ето как го четем в C#:

Program.cs 
using System.IO;  // Нужно за StreamReader!

static void Main()
{
    // Стъпка 1: Отваряме файла
    StreamReader sr = new StreamReader("data.txt");

    // Стъпка 2: Четем ред по ред
    string line1 = sr.ReadLine();  // "Мария Иванова"
    string line2 = sr.ReadLine();  // "Петър Георгиев"
    string line3 = sr.ReadLine();  // "Елена Димитрова"

    Console.WriteLine(line1);
    Console.WriteLine(line2);
    Console.WriteLine(line3);

    // Стъпка 3: Затваряме!
    sr.Close();
}
💡
Забележка: ReadLine() връща null когато файлът свърши. Така разбираме, че няма повече редове.

🔄 Пример 2: Четене с цикъл while

Когато не знаем колко реда има файлът, използваме цикъл:

Program.cs 
using System.IO;

static void Main()
{
    StreamReader sr = new StreamReader("data.txt");

    string line = sr.ReadLine();  // Четем първия ред

    while (line != null)          // Докато има редове
    {
        Console.WriteLine(line);   // Отпечатваме реда
        line = sr.ReadLine();       // Четем следващия
    }

    sr.Close();
}
✅
Шаблон: Този шаблон с while цикъл ще използвате постоянно! Запомнете го — той работи за файлове с произволен брой редове.

📋 Пример 3: ReadToEnd()

Ако искаме целия текст наведнъж (без да го четем ред по ред):

Program.cs 
using System.IO;

static void Main()
{
    StreamReader sr = new StreamReader("data.txt");

    string all = sr.ReadToEnd();  // Всичко наведнъж!
    Console.WriteLine(all);

    sr.Close();
}
📑
ReadLine()

Чете един ред. Подходящо за обработка ред по ред (търсене, броене, филтриране).

📜
ReadToEnd()

Чете целия файл наведнъж като един голям string. По-просто, но по-трудно за обработка.

🛡️ Бонус: using — по-безопасен начин

Какво става ако програмата гръмне с грешка преди да стигне до sr.Close()? Файлът остава заключен! 😱

Решението е using блок — той автоматично затваря StreamReader, дори при грешка:

Program.cs — по-безопасен вариант 
using System.IO;

static void Main()
{
    using (StreamReader sr = new StreamReader("data.txt"))
    {
        string line = sr.ReadLine();
        while (line != null)
        {
            Console.WriteLine(line);
            line = sr.ReadLine();
        }
    } // ← Тук sr се затваря АВТОМАТИЧНО!
}
Аналогия: Кран с таймер ⏱️

using блокът е като кран, който сам се затваря след определено време. Не можеш да забравиш да го затвориш — системата се грижи за теб!

🎯
Съвет: Използвайте using блока винаги, когато е възможно. Това е добра практика в C#!

🎓 Обобщение на урока

  • Потокът е връзката между програмата и файла — като тръба за данни
  • StreamReader чете текстови файлове. Винаги: отвори → прочети → затвори
  • ReadLine() чете по един ред. Връща null когато файлът свърши
  • ReadToEnd() чете целия файл като един string
  • using блокът гарантира, че StreamReader ще бъде затворен автоматично
